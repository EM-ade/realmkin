================================================================================
                          REWARD STRUCTURE
================================================================================

APY Rates (Annual Percentage Yield):
  - Flexible (no lock):    5% APY × 1.0x weight
  - 30-day lock:          12% APY × 1.5x weight
  - 90-day lock:          25% APY × 2.0x weight

Calculation Formula:
  Daily Rate = APY / 365 / 100
  Pending Rewards = Amount × Daily Rate × Days Staked × Weight

Example:
  1000 MKIN staked for 30 days at 12% APY with 1.5x weight:
  = 1000 × (12/365/100) × 30 × 1.5
  ≈ 14.79 MKIN rewards

  service cloud.firestore {
    match /databases/{database}/documents {
      match /users/{wallet} {
        allow read: if request.auth != null;
        allow write: if false;
      }
      match /stakes/{stakeId} {
        allow read: if request.auth != null;
        allow write: if false;
      }
      match /config/{document=**} {

================================================================================
                            HOW IT WORKS
================================================================================

STAKING FLOW:
  1. User connects wallet on /staking page
  2. User enters amount and selects lock period
  3. Frontend creates transaction to send tokens to staking wallet
  4. User signs transaction with wallet
  5. Frontend sends tx signature to POST /api/stake
  6. Backend verifies transaction on-chain
  7. Backend creates stake record in Firestore
  8. Frontend refreshes and shows confirmation

REWARD CALCULATION:
  1. Daily Cloud Function runs at midnight UTC
  2. Queries all active stakes from Firestore
  3. For each stake:
     - Calculates pending rewards using APY formula
     - Updates stake.rewards_earned
     - Updates user.total_rewards
  4. Process completes and logs results

CLAIMING REWARDS:
  1. User clicks "Claim Rewards" on /staking page
  2. Frontend calls POST /api/claim-rewards
  3. Backend calculates pending rewards
  4. Backend updates Firestore
  5. User sees updated reward balance

UNSTAKING FLOW:
  1. User clicks "Unstake" on /staking page
  2. Frontend checks if lock period has expired
  3. If locked: Show error "Still locked for X days"
  4. If unlocked: Call POST /api/unstake (action='initiate')
  5. Backend marks stake as "unstaking"
  6. Backend (admin) sends withdrawal transaction
  7. Backend calls POST /api/unstake (action='complete')
  8. User receives tokens in wallet

  Request: { wallet, stakeId, action, txSignature? }

  Response: { success, user, stakes, metrics }
